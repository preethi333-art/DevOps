## üìò GitHub Actions: Repository Variables & Secrets

---

### ‚úÖ What Are Variables and Secrets?

| Type         | Purpose                                                              | Visibility in UI   | Use With              |
| ------------ | -------------------------------------------------------------------- | ------------------ | --------------------- |
| **Variable** | Stores non-sensitive information (e.g., username, environment names) | ‚úÖ Visible/editable | `vars.VARIABLE_NAME`  |
| **Secret**   | Stores sensitive information (e.g., API keys, passwords)             | ‚ùå Hidden (masked)  | `secrets.SECRET_NAME` |

---

## üõ†Ô∏è Where to Add Variables and Secrets

1. **Go to your GitHub repository**
2. Click on **Settings ‚Üí Secrets and variables ‚Üí Actions**
3. You'll see:

   * `Repository secrets`
   * `Repository variables`
   * `Environment secrets` (used with `environments`, not covered in this session)

---

### üîë Naming Rules

* Use **alphanumeric characters and underscores**
* **Must not start with a number**
* Must **not start with `GITHUB_`** (reserved for GitHub's built-in variables)

---

## üîß Example: Add Variable and Secret

### üìå Variable

* **Name:** `USERNAME`
* **Value:** `interviewPro`

### üìå Secret

* **Name:** `PASSWORD`
* **Value:** `Test@1234` (or anything sensitive)

---

## üìÇ Sample Workflow: `environment-variable.yml`

```yaml
name: Access Repo Vars & Secrets

on:
  push:
    branches: [main]

jobs:
  demo:
    runs-on: ubuntu-latest
    steps:
      - name: Access Repository Variable
        run: echo "Username is ${{ vars.USERNAME }}"

      - name: Access Repository Secret
        run: echo "Password is ${{ secrets.PASSWORD }}"
```

---

## üß™ Output Behavior

| Step                       | Output                          |
| -------------------------- | ------------------------------- |
| Access Repository Variable | `Username is interviewPro`      |
| Access Repository Secret   | `Password is ********` (masked) |

> üõ°Ô∏è GitHub automatically masks secrets in logs to prevent accidental exposure.

---

## ‚ùå Common Mistakes

| Mistake                            | Reason                             |
| ---------------------------------- | ---------------------------------- |
| Typo in `vars.` or `secrets.` key  | Will result in empty output        |
| Trying to view secret values in UI | ‚ùå Not possible (by design)         |
| Using `GITHUB_` prefix             | ‚ùå Forbidden for user-defined names |

---

## ‚úÖ Summary

| Feature             | `vars`                      | `secrets`                    |
| ------------------- | --------------------------- | ---------------------------- |
| Type                | Non-sensitive               | Sensitive                    |
| Visibility in UI    | ‚úÖ Yes                       | ‚ùå No                         |
| Access in workflows | `${{ vars.VARIABLE_NAME }}` | `${{ secrets.SECRET_NAME }}` |
| Use cases           | Usernames, ENV flags, modes | Tokens, passwords, API keys  |
| Masked in logs      | ‚ùå No                        | ‚úÖ Yes                        |
